子元素未知宽高的水平垂直居中实现方式总结

2017.08.21 gruad HTML  热度 ℃

前言

关于水平垂直居中等一直以来都没有详细总结，都是遇到问题才开始去解决问题学习，关于布局中的水平垂直居中等，以前也总结过，可以参考把简单做好也不简单-css水平垂直居中
而自己总结的在github上面，但关于未知宽高的垂直居中还没有详细总结过，今天就总结一下，以备开发学习查阅

DOM结构

如未特殊说明DOM结构均为：

方法一：利用display:table

原理：
利用父标签为display:table，子标签dispaly:table;vertical-align:middle;（子元素与父元素的对准方式，控制垂直方向上对齐）text-align:center;（控制文字对其方式：水平方向对齐）实现
优点：父元素（parent）可以动态的改变高度（table元素的特性）
缺点：IE8以下不支持

结果：

方案二：利用transform:translate(-50%,-50%)

原理：和负margin:50%自身宽原理一样
优点：简单，思路清晰，同负-margin原理，易于理解
缺点：不支持IE9以下不支持transform属性，浏览器要求高caniuse查询浏览器是否支持该属性caniuse

结果同上方案一

方案三：使用css3 flex布局

原理：flex布局
优点：简单，快捷
缺点：不太兼容，浏览器要求高caniuse查询浏览器是否支持该属性caniuse

结果：

支持一下
